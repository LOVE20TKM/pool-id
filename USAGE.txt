LOVE20 Pool ID - Quick Reference
==================================

## Deployment

1. Set environment variables in .env file (copy from env.example)
2. Run deployment script:
   forge script script/DeployPoolID.s.sol:DeployPoolID --rpc-url $RPC_URL --broadcast

## Minting a Pool ID

1. Set POOL_NAME environment variable
2. Run minting script:
   forge script script/MintPoolIDExample.s.sol:MintPoolIDExample --rpc-url $RPC_URL --broadcast

## Querying Pool ID Information

Run query script with optional parameters:
   POOL_ID_ADDRESS=0x... forge script script/QueryPoolID.s.sol:QueryPoolID --rpc-url $RPC_URL

Optional parameters:
- TOKEN_ID: Query specific token
- POOL_NAME: Query specific pool name
- CALCULATE_FOR: Calculate mint cost for a name

## Mint Cost Formula

Base Cost = (LOVE20 maxSupply - totalSupply) / 10^8

For pool names:
- >= 10 bytes: Cost = Base Cost
- < 10 bytes: Cost = Base Cost * 10^(10 - byte_length)

Examples (assuming 21 billion LOVE20 max supply):
- 12 bytes: 210,000,000 LOVE20
- 10 bytes: 210,000,000 LOVE20
- 8 bytes:  21,000,000,000 LOVE20
- 6 bytes:  2,100,000,000,000 LOVE20
- 4 bytes:  210,000,000,000,000 LOVE20

## Testing

Run all tests:
   forge test

Run tests with gas report:
   forge test --gas-report

Run tests with detailed output:
   forge test -vvv

## Contract Structure

src/
├── interfaces/
│   └── IPoolID.sol          - Pool ID interface definition
├── PoolID.sol               - Main Pool ID contract (ERC721-based)

Dependencies:
- LOVE20 Core (@core/)       - Core protocol interfaces
- OpenZeppelin (@openzeppelin/) - Inherits ERC721 implementation from core/lib

test/
├── mocks/
│   └── MockLOVE20Token.sol  - Mock LOVE20 token for testing
└── PoolID.t.sol             - Comprehensive test suite

script/
├── DeployPoolID.s.sol       - Deployment script
├── MintPoolIDExample.s.sol  - Example minting script
└── QueryPoolID.s.sol        - Query script

## Key Features

✓ Inherits OpenZeppelin ERC721 standard implementation
✓ Unique pool names mapped to NFTs
✓ Dynamic pricing based on name length and LOVE20 supply
✓ Immutable pool names (cannot be changed after minting)
✓ Transferable ownership (via ERC721 standard)
✓ LOVE20 tokens locked in contract permanently (no withdrawal)
✓ Full test coverage with 18 test cases
✓ Simplified transfer mechanism (no IERC721Receiver callback)
✓ Battle-tested and audited OpenZeppelin code base
✓ No admin privileges or centralized control
